   ## Coding Guidelines

  1.)  Write clean, readable, and well-documented code

  2.)  Add docstrings for every public function

  3.)  Keep code modular and educational

  4.)  Follow PEP8 Python style guidelines

  5.)  Avoid breaking existing APIs

  6.)  Add pytest tests for new features when possible

## Running Tests
All tests use pytest.
To run the full test suite locally:
```bash
pip install pytest
pytest
```
Make sure all tests pass before submitting a pull request.
